Homework 2

  • Read chapter 3 in Core Java. From here on, don't expect me to tell you what to read. If you can't figure it out...
  • Repeat hw01, except in Java.
    • Add the ability to take command line arguments. The first argument, if provided should be the number of generations, the second, again only if provided, the name of the file to run.
    • Use the default values from hw01 if command line arguments are not provided. The default file, life.txt, should be assumed to be in the "current working directory", probably the directory your code is in.
    • Write your output to standard output, i.e. "the screen".
    • Do not put your code in a package or in a jar file. I assume if you do so, either you know more Java than you should if you are in this class, or you are totally dependent of an IDE that is packaging it all for you. If you are doing that, then you are likely not learning very much. When I want you to put code in either packages or jar files, I will tell you.
    • We have introduced all the API that you need. The various classes covered in the readings and in the slides. If you are using other classes, I expect that you have not been paying attention.
  • Remember:
    • First, let me say I know all these points will seem obvious to most anyone who has written code in almost any language, but still some students manage to surprise me. So, please be sure to pay attention to the following for this and future assignments.
    • Use good names. Do not use single letter names. Ok, i and j for loop variables. Maybe x and y for coordinates in a 2D space. But that's about it.
    • Remember Java's naming conventions.
    • Make good use of functions. Writing this entire program as a single function is a bad idea.
    • Provide reasonable comments
    • Use proper indentation. If you are using a decent editor, such as emacs, you editor should easily help you with indentation.
    • Always write clean, readable code.
    • You are writing Java code, the files should be .java files. Not .txt. And certainly not .docx! (Yes, someone once really did submit a Word file.)
    • All fields must be private. If all of your variables are "local" then you won't have to worry about this.
    • I addressed the issue of where your code should expect the default file, life.txt to be. But speaking more generally, do not assume I am using your choice of operating system. Also, I may not want to put my files where you want me to. In paricular do not hardcode absolute paths, such as C:\life.txt.
